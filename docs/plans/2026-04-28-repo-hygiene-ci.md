### Task 3: Remove hardcoded personal paths

**Files:**
- Modify: `packages/codegraph/src/mcp/http-server.ts:54`
- Modify: `packages/codegraph/src/dashboard/server.ts:14`
- Modify: `packages/codegraph/src/mcp/proxy.ts:28`

- Modify: `packages/codegraph/src/mcp/tools/sessions.ts:99,172`
- Modify: `packages/codegraph/src/mcp/tools/memory.ts:74`
- Modify: `packages/codegraph/src/mcp/tools/skills.ts:130,174`
- Modify: `packages/codegraph/src/mcp/tools/components.ts:332,379`

Two categories of hardcoding:

**Category A — Personal Mac path (MUST fix):**
- `http-server.ts:54` → `SKILLBRAIN_ROOT` fallback to `/Users/dan/...`
- `dashboard/server.ts:14` → same

Fix: replace fallback with `process.cwd()` (safe — Docker sets WORKDIR, local dev uses cwd).

**Category B — `memory.fl1.it` URLs in tool descriptions and email templates:**
These are UX strings shown to users. They should use the configurable `PUBLIC_URL` env var so each deployment shows its own URL.

Fix: extract a `dashboardUrl()` helper that reads `PUBLIC_URL` env var with `memory.fl1.it` as last-resort fallback. Use it in all tool response strings and email templates.

**Step 1: Fix SKILLBRAIN_ROOT fallback in both files**

In `packages/codegraph/src/mcp/http-server.ts:54`:
```typescript
// BEFORE:
const SKILLBRAIN_ROOT = process.env.SKILLBRAIN_ROOT || '/Users/dan/Desktop/progetti-web/MASTER_Fullstack session'
// AFTER:
const SKILLBRAIN_ROOT = process.env.SKILLBRAIN_ROOT || process.cwd()
```

In `packages/codegraph/src/dashboard/server.ts:14`:
```typescript
// BEFORE:
const SKILLBRAIN_ROOT = process.env.SKILLBRAIN_ROOT || '/Users/dan/Desktop/progetti-web/MASTER_Fullstack session'
// AFTER:
const SKILLBRAIN_ROOT = process.env.SKILLBRAIN_ROOT || process.cwd()
```

**Step 2: Create dashboardUrl helper in constants.ts**

Append to `packages/codegraph/src/constants.ts`:
```typescript
// ── Public URL ──
/** Base URL for dashboard links in MCP tool responses and emails. */
export function dashboardUrl(): string {
  return (process.env.PUBLIC_URL || process.env.OAUTH_ISSUER || 'https://memory.fl1.it').replace(/\/+$/, '')
}
```

**Step 3: Use dashboardUrl() in http-server.ts**

Replace hardcoded `memory.fl1.it` in:
- `PUBLIC_ISSUER` fallback (line 58): `const PUBLIC_ISSUER = process.env.OAUTH_ISSUER || process.env.PUBLIC_URL || dashboardUrl()`
- `SMTP_FROM` (line 63): keep as-is (email domain is deployment-specific, fine as env var)
- Email invite template (lines 102, 107): replace `https://memory.fl1.it/` with `${dashboardUrl()}/` and `${dashboardUrl()}/mcp`

**Step 4: Use dashboardUrl() in MCP tool response strings**

In each tool file, import `dashboardUrl` from constants and replace hardcoded URLs:
- `sessions.ts:99` → `` `${dashboardUrl()}/#/design-systems` ``
- `sessions.ts:172` → `` `${dashboardUrl()}/#/review` ``
- `memory.ts:74` → `` `${dashboardUrl()}/#/review` ``
- `skills.ts:130,174` → `` `${dashboardUrl()}/#/review` ``
- `components.ts:332,379` → `` `${dashboardUrl()}/#/design-systems` ``

### Task 4: Add GitHub Actions CI

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Create the workflow file**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: packages/codegraph

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: packages/codegraph/pnpm-lock.yaml

      - run: pnpm install --frozen-lockfile

      - name: Type check
        run: pnpm exec tsc --noEmit

      - name: Build
        run: pnpm run build

      - name: Test
        run: pnpm test
```

**Step 2: Verify workflow syntax**

```bash
# Quick YAML validation
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo "YAML OK"
```

**Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions pipeline (build + typecheck + test)"
```
